Implementation Prompt

This is the exact prompt I used across all 11 platforms:

Click to view full prompt 
PROJECT: #30DayMapChallenge 2025 - Day 1: POINTS

CONCEPT:
Create an interactive map showing all global earthquakes from the past 30 days. Each earthquake is displayed as a point sized by magnitude (larger = stronger earthquake) and colored by depth (darker = deeper underground). Include hover tooltips that show magnitude, location name, date/time, and depth. The map should reveal natural patterns like the Pacific "Ring of Fire."

REQUIREMENTS:

Visual:
- Interactive web-based map (flat or globe projection, your choice)
- Dark or light basemap that makes colored points stand out
- Modern, clean design with clear visual hierarchy
- Responsive layout (works on mobile and desktop)
- Points should be clearly visible and distinguishable
- Color gradient should be intuitive (e.g., red/orange for shallow, yellow for medium, blue for deep)

Data:
- Data source: USGS Earthquake API (free, no API key required)
- Endpoint: https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_month.geojson
- Format: GeoJSON (standard format, works with all mapping libraries)
- Fields to use:
  - properties.mag - magnitude (for point size)
  - properties.depth - depth in kilometers (for point color)
  - properties.place - location name (for tooltip)
  - properties.time - timestamp (for tooltip, convert to readable date)
  - geometry.coordinates - [longitude, latitude] array

Functionality:
- Zoom and pan controls
- Hover tooltips showing: magnitude, location name, date/time, depth
- Points sized proportionally by magnitude (use minimum and maximum sizes, e.g., 3px to 20px radius)
- Points colored by depth (use a color gradient, e.g., red/orange for shallow < 50km, yellow for medium 50-300km, blue for deep > 300km)
- Smooth interactions (no lag when hovering/zooming)
- Optional: Click to show more details in a popup

Technical:
- Single HTML file preferred (unless platform requires different structure)
- Use modern JavaScript (ES6+)
- Mapping library: Your choice (Leaflet, Mapbox GL JS, MapLibre GL JS, D3.js, or platform's preferred library)
- Fetch the GeoJSON from the USGS API endpoint
- Handle the data and render points efficiently
- Use CDN links for any external libraries
- No build step required - should run directly

SPECIFIC TO DAY 1 (POINTS THEME):
- Focus on interesting point symbolization (size + color encoding)
- Demonstrate density visualization (where earthquakes cluster)
- Show how point maps reveal geographic patterns
- Make it visually striking for social media sharing

DELIVERABLES:
- Fully functional interactive earthquake map
- Title: "Global Earthquakes - Last 30 Days"
- Attribution: "Data: USGS Earthquake Hazards Program"
- Legend showing magnitude scale (size) and depth scale (color)
- Brief description: "Earthquakes from the past month. Point size = magnitude. Color = depth."
- Clean, commented code
- Responsive and works in modern browsers

Key Insights

‚úÖ What Worked
  • Same prompt across all platforms proved the concept - one implementation prompt, 11 different outputs
  • Fast one-shot platforms: Bolt, Lovable, MagicPatterns, Base44 all delivered working code in ~5 min
  • Mobile responsive design winner - Lovable excels on mobile with great responsive design, making it the best overall considering mobile-first usage (moved to #1 ranking)
  • Top 3 rankings - Lovable (#1 - best mobile + desktop balance), ChatGPT (#2 - best desktop UI but mobile issues), Bolt (#3 - best data handling but mobile space issues)
  • Tooltip quality matters - Lovable, Bolt, ChatGPT had strong tooltips; others lagged behind
  • USGS API was a good choice - standard GeoJSON works everywhere (where platforms allowed)
‚ùå What Didn't Work
  • Mobile responsive issues - ChatGPT Canvas has imbalanced header and overlapping UI elements; Bolt.new's legend takes up half the screen (needs collapsible or better spacing); Base44 legend overlays too much; MagicPatterns and Google AI Studio have alignment/fitting issues
  • Claude & Gemini Artifacts struggled with external API integration - will retry them in future days since they work sometimes
  • CSP/CORS issues are the biggest failure mode for artifact-style platforms
  • Firebase Studio's API key friction - requiring 2 keys (Gemini + Google Maps) slows initial setup
  • V0.dev needed iteration despite being highly regarded - not always one-shot success; also map area too small on mobile
Surprises
  • Base44 - Unknown platform with the best stats dashboard and dark theme
  • Firebase Studio's code editor - Most full-featured editor experience (VS Code in browser), though others like Bolt and Lovable also have code editing
  • MagicPatterns - Unknown platform delivered well, tested 2 variants (Tailwind + Chakra UI), both worked
  • Google AI Studio - Quietly delivered solid implementation without fanfare
  • One-shot AI generation works - Most platforms delivered functional maps from a single prompt in ~5 minutes, good for rapid prototyping
The Big Learning

The real success: Previously, it would take me WAY longer to build WORSE versions of ANY of these maps. Now, I built 11 in one sitting. Plus a custom page to talk about it. And this is just the starting point.

Most platforms delivered functional, interactive earthquake maps from a single prompt. ChatGPT, Bolt, Lovable, AI Studio, Firebase Studio, MagicPatterns, v0, and Base44 all generated working implementations in ~5 minutes with zero iteration. That's 11 different interactive maps created in one session. Previously, I might not have been able to build even one of these from scratch. That's the real breakthrough.

These AI-generated maps are helpful quick win starting points. They're inspiring, they work, and they demonstrate the core concept. But the next step, making them truly useful and production-ready, adding a backend and custom features, refining UX, handling edge cases...that's where human design and engineering takes over. And that's the cool part: AI gives you the foundation in minutes, freeing us to focus on what we want to do.

This is just a starting point. Use quick AI generation as your rapid prototyping superpower. Get ideas working fast, then iterate and refine with intention. When mobile matters (which it always should), Lovable is the current winner for responsive foundations. But remember‚Äîthe ability to create 11 working maps in one sitting? That's the new baseline, and there's so much more to build on top of it.

Editorial

Day 1 worked well. The parallel testing of 10+ platforms revealed patterns that would be hard to see testing one at a time. Overall, maps and data showed up on most, some didn't work, and mobile responsive design became a factor in the rankings.

The Top 3: After mobile evaluation, Lovable ranks #1 - excellent on both desktop and mobile, with great responsive design making it the best overall choice. ChatGPT Canvas ranks #2 - most polished desktop UI with good permission model, but mobile has overlapping elements that detract from the experience. Bolt.new ranks #3 - handles data well with strong stats display, but legend takes up too much mobile screen space. Mobile performance matters significantly for map accessibility, and Lovable is the only top performer that excels on both platforms.

The Mobile Reality Check: Testing responsive design revealed that desktop polish doesn't always translate to mobile excellence. ChatGPT Canvas and Bolt.new are clearly A-tier on desktop, but their mobile implementations have noticeable issues (overlapping UI, space management problems). This shifted Lovable from "tied for first" to "clearly first" - its mobile experience is genuinely excellent while maintaining strong desktop performance. For maps that need to work everywhere, mobile responsiveness is non-negotiable, and only Lovable delivered that across the top tier.

The Surprises: Base44 has the best stats dashboard with a clean dark theme, though tooltips lag behind the top 3. Firebase Studio offers the most full-featured code editor (VS Code in browser) for developers who want both AI generation and full code control. MagicPatterns tested both Tailwind and Chakra UI variants - both worked.

The Disappointment: Claude Artifacts and Gemini Artifacts both struggled with external APIs. These are strong chat models for ideation, but their sandbox restrictions limit real web development. Interesting that Google AI Studio Apps (using the same Gemini model) succeeded where Gemini Artifacts failed - the platform matters more than the model.

The Prototyping Revolution: The real takeaway from Day 1 isn't which platform is best‚Äîit's that previously, it would take me WAY longer to build WORSE versions of ANY of these maps. Now, I built 11 in one sitting. Plus a custom page to talk about it. And this is just the starting point.. Most platforms delivered functional, interactive maps in ~5 minutes from a single prompt. That's the huge success: turning something that was previously difficult or impossible into something achievable in a single session. These aren't production-ready systems, but they're good starting points. They're inspiring, they demonstrate the concept, and they give you a foundation to build on. The next steps‚Äîcustomization, robustness, uniqueness‚Äîthat's where the real engineering begins. And that's exciting: AI handles the boilerplate, freeing you to focus on what makes your project special. But more importantly, this is just the starting point‚Äîthe ability to rapidly prototype multiple working solutions in parallel opens up entirely new workflows.

Looking ahead: Day 1 established clear patterns. Mobile-responsive design will be a key evaluation criteria going forward - desktop-only excellence isn't enough. For Day 2 and beyond, I'll focus on the proven performers (especially Lovable for mobile-critical projects) and retry Claude/Gemini Artifacts on different use cases. Firebase Studio is worth the API key setup when Firebase features are needed. But most importantly, I'll continue exploring how one-shot prompting can accelerate prototyping while keeping an eye on what comes next‚Äîmaking these prototypes production-ready.
